# 瞬間作文トレーニング 例文作成指示書

## 1. 概要

この指示書は、韓国語学習アプリ「瞬間作文トレーニング」機能の例文データを作成するためのものです。

### 1.1 瞬間作文とは

日本語の文を見て、瞬時に韓国語に翻訳して発話する練習方法です。ユーザーは音声入力で回答し、正誤判定を受けます。

### 1.2 作成する内容

- 各文法項目ごとに **10問** の例文を作成
- 日本語文と、それに対応する韓国語文のペア
- 必要に応じて別解（alternativeAnswers）も用意

---

## 2. 対象文法項目

文法項目は `grammar_index.json` に定義されています。各カテゴリと項目数は以下の通りです：

| カテゴリID | カテゴリ名 | 項目数 |
|------------|------------|--------|
| `orthography` | 表記・発音 | 22項目 |
| `substantive` | 体言 | 51項目 |
| `particle` | 助詞 | 25項目 |
| `conjugation` | 用言と活用 | 22項目 |
| `sentence_ending` | 終止形と待遇法 | 16項目 |
| `connective` | 接続形 | 23項目 |
| `adnominal` | 連体形 | 14項目 |
| `tense_aspect` | 時制 | 41項目 |
| `expression` | さまざまな表現 | 45項目 |
| `quotation` | 直接話法と間接話法 | 11項目 |
| `word_formation` | 単語の作り | 30項目 |

**総計: 300項目 × 10問 = 3,000問**

---

## 3. 出力形式（JSON）

### 3.1 ファイル構造

```
application/assets/quick_translation/
├── particle/
│   ├── grm_particle_001.json
│   ├── grm_particle_002.json
│   └── ...
├── connective/
│   ├── grm_connective_001.json
│   └── ...
└── ...
```

### 3.2 JSONフォーマット

```json
{
  "grammarRef": "grm_particle_001",
  "grammarTitle": "이/가",
  "grammarSubtitle": "主格助詞（〜が）",
  "category": "particle",
  "questions": [
    {
      "id": "qt_particle_001_01",
      "order": 1,
      "japanese": "猫がいます。",
      "korean": "고양이가 있어요.",
      "alternativeAnswers": ["고양이가 있습니다."],
      "hint": "고양이...",
      "explanation": "「고양이」は母音で終わるので「가」を使います。"
    },
    {
      "id": "qt_particle_001_02",
      "order": 2,
      "japanese": "本があります。",
      "korean": "책이 있어요.",
      "alternativeAnswers": ["책이 있습니다."],
      "hint": "책...",
      "explanation": "「책」はパッチムがあるので「이」を使います。"
    }
    // ... 合計10問
  ]
}
```

### 3.3 フィールド説明

| フィールド | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `grammarRef` | String | ✓ | 文法辞典の項目ID（例: `grm_particle_001`） |
| `grammarTitle` | String | ✓ | 文法項目のタイトル（韓国語、例: `이/가`） |
| `grammarSubtitle` | String | ✓ | 文法項目の日本語説明（例: `主格助詞（〜が）`） |
| `category` | String | ✓ | 文法カテゴリID（例: `particle`） |
| `questions` | Array | ✓ | 問題リスト（必ず10問） |

#### questions配列の各要素

| フィールド | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `id` | String | ✓ | 問題の一意識別子（形式: `qt_{category}_{itemNumber}_{questionNumber}`） |
| `order` | Number | ✓ | 問題の順番（1〜10） |
| `japanese` | String | ✓ | 日本語の問題文 |
| `korean` | String | ✓ | 正解の韓国語文 |
| `alternativeAnswers` | Array | - | 別解のリスト（複数の正解がある場合） |
| `hint` | String | - | ヒント（韓国語の最初の数文字） |
| `explanation` | String | - | 解説（なぜその答えになるか） |

## 4. 例文作成のルール

### 4.1 語彙レベルの制限（重要）

**TOPIK初級〜中級レベルの語彙のみを使用してください。**

#### 使用してよい語彙の例

**名詞:**
- 人: 사람, 친구, 가족, 학생, 선생님, 아이, 부모님, 형, 누나, 동생
- 場所: 집, 학교, 회사, 병원, 식당, 카페, 공원, 역, 가게, 도서관
- 物: 책, 가방, 휴대폰, 컴퓨터, 음식, 물, 커피, 차, 옷, 신발
- 時間: 오늘, 내일, 어제, 지금, 아침, 점심, 저녁, 주말, 방학
- 抽象: 일, 공부, 운동, 여행, 영화, 음악, 사진, 이야기, 생각, 계획

**動詞:**
- 가다, 오다, 먹다, 마시다, 자다, 일어나다, 보다, 듣다, 읽다, 쓰다
- 만나다, 기다리다, 말하다, 묻다, 대답하다, 알다, 모르다, 생각하다
- 사다, 팔다, 주다, 받다, 보내다, 찾다, 잃다, 놓다, 들다, 열다
- 시작하다, 끝나다, 도착하다, 출발하다, 걷다, 달리다, 타다, 내리다

**形容詞:**
- 좋다, 나쁘다, 크다, 작다, 많다, 적다, 높다, 낮다, 길다, 짧다
- 덥다, 춥다, 맛있다, 맛없다, 재미있다, 재미없다, 어렵다, 쉽다
- 바쁘다, 피곤하다, 행복하다, 슬프다, 기쁘다, 아프다, 건강하다

**副詞:**
- 아주, 매우, 정말, 너무, 조금, 많이, 빨리, 천천히, 잘, 못
- 자주, 가끔, 항상, 보통, 먼저, 나중에, 다시, 또, 함께, 같이

#### 使用を避ける語彙

以下のような難しい単語は使用しないでください：

- 漢字語の専門用語: 취약하다, 심각하다, 획기적이다, 효율적이다
- 抽象度の高い名詞: 현상, 본질, 맥락, 개념, 인식
- ビジネス・学術用語: 분석하다, 검토하다, 수립하다, 시행하다
- 日常会話で使わない形容詞: 광활하다, 심오하다, 현저하다
- 文語体の表現: 그러하다, 이러하다, 여하튼

### 4.2 文の長さ

- **短く簡潔に**: 5〜20文字程度の韓国語文
- **1文で完結**: 複数の文に分けない
- **音声入力で答えやすい長さ**: 息継ぎなしで言える程度

#### 良い例
- 고양이가 있어요. (8文字)
- 오늘 날씨가 좋아요. (10文字)
- 친구를 만나서 영화를 봤어요. (15文字)

#### 悪い例
- 어제 친구를 만나서 같이 영화관에 가서 재미있는 영화를 보고 맛있는 저녁을 먹었어요. (長すぎる)

### 4.3 文体

- **해요体（-아요/-어요）を基本**とする
- **합니다体（-습니다/-ㅂ니다）は別解**として追加
- 반말（パンマル）は使用しない

```json
{
  "korean": "학교에 가요.",
  "alternativeAnswers": ["학교에 갑니다."]
}
```

### 4.4 alternativeAnswers（別解）の追加基準

以下の場合に別解を追加してください：

1. **해요体と합니다体の両方が自然な場合**
   ```json
   "korean": "밥을 먹어요.",
   "alternativeAnswers": ["밥을 먹습니다."]
   ```

2. **同義語で置き換え可能な場合**
   ```json
   "korean": "뭐가 있어요?",
   "alternativeAnswers": ["무엇이 있어요?"]
   ```

3. **語順の入れ替えが可能な場合**
   ```json
   "korean": "나는 학교에 가요.",
   "alternativeAnswers": ["학교에 나는 가요."]
   ```

4. **助詞の省略が自然な場合**
   ```json
   "korean": "저는 학생이에요.",
   "alternativeAnswers": ["저 학생이에요."]
   ```

### 4.5 hintの形式

- 韓国語の正解文の**最初の単語または最初の2〜3文字**
- 末尾に「...」を付ける

```json
"korean": "고양이가 있어요.",
"hint": "고양이..."
```

```json
"korean": "오늘 날씨가 좋아요.",
"hint": "오늘..."
```

### 4.6 explanationの書き方

- **なぜその文法を使うか**を簡潔に説明
- 日本語で記述
- 50文字程度に収める

```json
"explanation": "「책」はパッチムがあるので「이」を使います。"
```

```json
"explanation": "理由を表す「-아서/어서」は原因・理由を示します。"
```

## 5. カテゴリ別の注意事項

### 5.1 表記・発音（orthography）

発音変化を意識した例文を作成してください。

```json
// 連音化の例
{
  "japanese": "日本語を勉強します。",
  "korean": "일본어를 공부해요.",
  "explanation": "「일본어」は連音化で[일보너]と発音します。"
}
```

### 5.2 体言（substantive）

代名詞・数詞・疑問詞などを使った例文を作成してください。

```json
// 指示代名詞の例
{
  "japanese": "これは本です。",
  "korean": "이것은 책이에요.",
  "alternativeAnswers": ["이건 책이에요."],
  "explanation": "「이것」は「これ」、「은」は主題を示します。"
}
```

### 5.3 助詞（particle）

助詞の使い分けがポイントになる例文を作成してください。

```json
// 이/가 と 은/는 の違い
{
  "japanese": "誰が来ましたか？",
  "korean": "누가 왔어요?",
  "explanation": "疑問詞「誰」と使う場合は「이/가」を使います。"
}
```

### 5.4 用言と活用（conjugation）

不規則活用を含む例文を作成してください。

```json
// ㅂ不規則
{
  "japanese": "今日は暑いです。",
  "korean": "오늘은 더워요.",
  "explanation": "「덥다」はㅂ不規則で「더워요」になります。"
}
```

### 5.5 終止形と待遇法（sentence_ending）

様々な終止形を使った例文を作成してください。

```json
// -네요（感嘆）
{
  "japanese": "わあ、きれいですね。",
  "korean": "와, 예쁘네요.",
  "explanation": "「-네요」は感嘆や新しい発見を表します。"
}
```

### 5.6 接続形（connective）

接続表現を使った例文を作成してください。

```json
// -아서/어서（理由）
{
  "japanese": "疲れたので休みます。",
  "korean": "피곤해서 쉬어요.",
  "explanation": "「-아서/어서」は理由を表します。"
}
```

### 5.7 連体形（adnominal）

名詞を修飾する形の例文を作成してください。

```json
// -는（現在連体形）
{
  "japanese": "勉強している学生",
  "korean": "공부하는 학생",
  "explanation": "動詞の現在連体形は「-는」を使います。"
}
```

### 5.8 時制（tense_aspect）

時制・否定・受身・使役などの例文を作成してください。

```json
// 過去形
{
  "japanese": "昨日映画を見ました。",
  "korean": "어제 영화를 봤어요.",
  "alternativeAnswers": ["어제 영화를 봤습니다."],
  "explanation": "過去形は「-았/었」を付けます。"
}
```

### 5.9 さまざまな表現（expression）

慣用表現を使った例文を作成してください。

```json
// -고 싶다（願望）
{
  "japanese": "韓国に行きたいです。",
  "korean": "한국에 가고 싶어요.",
  "explanation": "「-고 싶다」は願望を表します。"
}
```

### 5.10 直接話法と間接話法（quotation）

引用表現を使った例文を作成してください。

```json
// -다고 하다（平叙文の間接引用）
{
  "japanese": "友達が来ると言いました。",
  "korean": "친구가 온다고 했어요.",
  "explanation": "「-ㄴ/는다고 하다」は平叙文の間接引用です。"
}
```

### 5.11 単語の作り（word_formation）

副詞形・体言形などの例文を作成してください。

```json
// -게（副詞形）
{
  "japanese": "速く走ります。",
  "korean": "빠르게 달려요.",
  "explanation": "「-게」は形容詞を副詞化します。"
}
```

## 6. 品質チェックリスト

例文作成後、以下の項目を確認してください：

### 6.1 基本チェック

- [ ] 1ファイルにつき必ず10問あるか
- [ ] `id`の形式は正しいか（`qt_{category}_{itemNumber}_{questionNumber}`）
- [ ] `order`は1〜10まで連番になっているか
- [ ] 日本語と韓国語の意味が正確に対応しているか

### 6.2 語彙チェック

- [ ] TOPIK高級レベルの難しい単語を使っていないか
- [ ] 日常会話で使わない専門用語を使っていないか
- [ ] 学習者が知らないと思われる固有名詞を使っていないか

### 6.3 文法チェック

- [ ] その文法項目を使った例文になっているか
- [ ] 韓国語として自然な文になっているか
- [ ] スペルミスがないか

### 6.4 形式チェック

- [ ] 文の長さは適切か（5〜20文字程度）
- [ ] 해요体が基本になっているか
- [ ] 必要な場合に別解を追加しているか
- [ ] ヒントは正解の最初の部分になっているか

## 7. 作成手順

### Step 1: 文法項目の確認

対象の文法項目のJSONファイルを読み、以下を確認してください：

- `title`: 文法項目名（韓国語）
- `subtitle`: 日本語説明
- `content.summary`: 文法の概要
- `content.usageCases`: 使用例
- `content.formationRules`: 活用ルール

### Step 2: 例文の設計

10問の例文を設計します。以下のバランスを意識してください：

- **基本的な使い方**: 3〜4問（最もシンプルな使い方）
- **やや応用的な使い方**: 3〜4問（少し複雑な文脈）
- **よくある間違いを意識した問題**: 2〜3問（学習者が間違えやすいポイント）

### Step 3: JSON作成

テンプレートに沿ってJSONファイルを作成してください。

### Step 4: 品質チェック

チェックリストに従って確認してください。

## 11. よくある質問

### Q1: 文法詳細ファイルの例文をそのまま使ってもいいですか？

A: 参考にするのはOKですが、そのままコピーするのではなく、瞬間作文用に適した形（日本語→韓国語の翻訳形式）にアレンジしてください。

### Q2: 10問より多く作ってもいいですか？

A: いいえ、必ず10問ちょうどにしてください。システムが10問固定で動作するように設計されています。

### Q3: 別解はいくつまで追加できますか？

A: 技術的な制限はありませんが、2〜3個程度が適切です。あまり多すぎると管理が大変になります。

### Q4: 日本語の文は敬体（です・ます）で統一すべきですか？

A: 基本的には敬体で統一してください。ただし、韓国語の반말を学ぶ項目では、日本語もカジュアルな表現を使ってOKです。

### Q5: 韓国語のスペースはどうすればいいですか？

A: 韓国語の正書法に従ってください。ただし、音声認識では若干の揺れがあるため、スペースの有無は判定時に許容されます。
